# Codebase Explanation — ArbMatrix

This document explains the purpose and contents of the files and folders in this repository. It is intended to help new developers quickly understand where key logic lives and how the app is structured.

---

## Project summary
ArbMatrix is a React + Vite single-page app that visualizes arbitrage opportunities between custom units (currencies/assets). The core algorithmic logic uses the Bellman-Ford algorithm (in `src/lib/algorithms.js`) to detect negative cycles when converting rates are transformed via -ln(rate). The UI uses React + Zustand for state management and Framer Motion for transitions.

## `src/` — source code

This folder contains the complete React app source.

- `src/main.jsx`
  - Entry point that mounts the React app to the DOM using `createRoot` and renders `<App />` inside `StrictMode`.

- `src/App.jsx`
  - Top-level application component. Responsibilities:
    - Imports global state from `src/store/useAppStore` and algorithm helpers from `src/lib/algorithms.js`.
    - Maps `activePage` to page components (Dashboard, Matrix, Arbitrage, etc.).
    - Lays out the main structure: `Sidebar`, `TickerTape`, `Header`, and the page content area (with animated transitions via `framer-motion`).
    - Computes arbitrage opportunities using `computeArbitrageOpportunities(units, rates, fees)` and passes counts to the `Sidebar`.

- `src/assets/`
  - Static assets used by the UI (logos, images). Inspect for any image references.

- `src/components/`
  - `layout/` (`src/components/layout/index.jsx`)
    - Exports layout-related components used by `App.jsx`: `Sidebar`, `Header`, `TickerTape`, and also a set of inline icon components (`Icons`) used across the app.
    - `Sidebar` renders navigation groups, active item gradient, and a course info footer.
    - `TickerTape` shows a horizontal scrolling list of live rates pulled from the global store.
    - `Header` shows the page title, search placeholder, add-unit modal (with `Modal`, `Input`, and `Button` from `ui`), notifications and user avatar.
  - `ui/` (`src/components/ui/index.jsx`)
    - Reusable UI primitives and design tokens used across the app:
      - `T` (design tokens: colors), `SHADOW`, `GRADIENT` constants
      - `SectionTitle`, `AnimatedNumber`, `StatCard`, `Badge`, `ConfidenceBadge`, `StatusBadge`, `Button`, `Input`, `Card`, `EmptyState`, `NotificationToast`, `Spinner`, `Modal`, `ProgressBar`, `Divider`.
    - These components are lightweight, style-driven, and use inline styles and `framer-motion` for animation.

- `src/pages/`
  - Contains one file per major page/route in the app:
    - `AnalyticsPage.jsx` — analytics and charts page.
    - `ArbitragePage.jsx` — lists detected arbitrage loops and details.
    - `CustomUnitsPage.jsx` — manage custom units and presets.
    - `DashboardPage.jsx` — the main overview dashboard (charts, summary).
    - `GraphExplorerPage.jsx` — visual exploration of the conversion graph.
    - `LoopVisualizerPage.jsx` — step-through visualizer for a single arbitrage loop.
    - `MatrixPage.jsx` — adjacency matrix editor for rates.
    - `SettingsPage.jsx` — app preferences and workspace config.
    - `SimulatorPage.jsx` — simulate compounding or running a loop.
  - Each page imports UI primitives and hooks from the store and `lib` as needed.

- `src/lib/algorithms.js`
  - Core algorithmic logic for the app.
  - Exports:
    - `bellmanFord(units, rates)` — transforms rates with -ln(rate) and runs Bellman-Ford to detect negative cycles. Returns raw cycles (path + gross multiplier + profit).
    - `computeArbitrageOpportunities(units, rates, fees)` — augments `bellmanFord` output with fee simulation, risk scoring, confidence, status, and sorted results.
    - `simulateLoop(loop, initialAmount, rates, fees)` — simulates running a single loop with fees and returns step-by-step details plus ROI.
    - `buildMatrix(units, rates)` — returns a 2D matrix representation (useful for `MatrixPage`).
    - `countActivePaths(units, rates)` — counts active rate paths.
    - `fmt(value, precision)` and `heatColor(value, min, max)` — helpers for formatting and heatmap color selection.

- `src/store/useAppStore.js`
  - Zustand-based global state store with persistence (via `persist` middleware).
  - Exports `DEFAULT_UNITS`, `DEFAULT_RATES`, `DEFAULT_UNIT_META` constants and the store hook `useAppStore`.
  - Key state values:
    - `units`, `rates`, `unitMeta`, `fees`, `precision`, `simHistory`, `notifications`, `selectedLoop`, `activePage`, `rateHistory`.
  - Key actions:
    - `setPage`, `addUnit`, `removeUnit`, `updateUnitMeta`, `setRate`, `removeRate`, `setFees`, `setPrecision`, `addSimResult`, `clearSimHistory`, `setSelectedLoop`, `_notify`, `dismissNotification`, `importWorkspace`, `exportWorkspace`, `resetWorkspace`.
  - Persistence options: partialize to persist units, rates, unitMeta, fees, precision, simHistory, rateHistory under name `arbmatrix-v3`.

## How to run

Prerequisites: Node.js (v18+ recommended), npm installed.

1. Install dependencies:

```bash
npm install
```

2. Start dev server (Vite):

```bash
npm run dev
```

3. Build for production:

```bash
npm run build
```

4. Preview production build locally:

```bash
npm run preview
```

## Notes and pointers

- The UI uses inline styles heavily and depends on design tokens in `src/components/ui/index.jsx` (`T`, `SHADOW`, `GRADIENT`). Changing these tokens updates the look app-wide.
- The arbitrage detection uses mathematical transforms that are sensitive to numerical precision; `src/store/useAppStore.js` exposes a `precision` setting used across the UI.
- The `src/lib/algorithms.js` file contains both detection and utility helpers — if you split responsibilities later, consider separating detection (bellmanFord) from simulation (simulateLoop).
- Zustand is used for a small and simple global store; `persist` writes to localStorage by default in browsers.

## Per-component props (quick reference)

Below are the most important components developers will interact with, their main props, expected types and small usage notes. This is not exhaustive but covers the practical surface area.

### Layout components (`src/components/layout/index.jsx`)

- `Icons` (collection of inline SVG components)
  - Usage: `<Icons.Dashboard />`, `<Icons.Zap />` — no props. Small, self-contained SVGs used across nav and buttons.

- `TickerTape()`
  - Props: none (reads `units` and `rates` from the store)
  - Returns: a scrolling ticker element. If there are no items it returns `null`.
  - Note: The ticker clones items to create a continuous scroll animation.

- `Sidebar({ opportunities })`
  - Props:
    - `opportunities` (number) — count shown as a badge for the Arbitrage nav item.
  - Behavior: reads `activePage` and `setPage` from the store to render navigation and change pages.

- `Header()`
  - Props: none (reads actions and state from store)
  - Behavior: renders page title, search placeholder, add-unit modal. Uses `Modal`, `Input`, `Button` from `ui`.
  - Internal callbacks:
    - `addUnit(form.name, meta)` — calls store action to create a unit.

### UI primitives (`src/components/ui/index.jsx`)

These primitives are small, self-contained components used throughout pages.

- `SectionTitle({ color, icon, children, action, style })`
  - `color` string — accent color (default `#2563EB`).
  - `icon` React node — optional small chip.
  - `children` React node — title text.
  - `action` React node — optional element on the right (e.g., a button).

- `AnimatedNumber({ value, decimals, prefix, suffix, duration })`
  - `value` number — target numeric value.
  - `decimals` number — decimal precision (default 2).
  - `prefix` / `suffix` strings — optional.
  - `duration` ms — animation duration (default 700).

- `StatCard({ label, value, sub, color, icon, pulse })`
  - `label` string — small uppercase label.
  - `value` string|number — prominent number.
  - `sub` string — small secondary text.
  - `color` string — accent color.
  - `icon` node — small icon on the right.
  - `pulse` boolean — enables glow animation.

- `Badge({ children, color, glow })`
  - Simple inline badge. `color` accepts token values from `T`.

- `ConfidenceBadge({ level })`
  - `level` one of `HIGH`, `MED`, `LOW` — maps to a color and optional glow.

- `StatusBadge({ status })`
  - `status` values: `profitable`, `marginal`, `eliminated` — renders labeled badge.

- `Button({ children, variant, onClick, disabled, style, size })`
  - `variant`: `primary | ghost | danger | success | accent`.
  - `size`: `sm | md | lg` (default `md`).

- `Input({ label, ...props })`
  - `label` string — above the input.
  - Pass standard input props (`value`, `onChange`, `placeholder`, etc.).

- `Card({ children, style, hover, glowColor })`
  - Generic surface container with optional hover effect.

- `EmptyState({ icon, title, body, action })`
  - Full-card empty state used by pages when there's no data.

- `NotificationToast({ notifications, onDismiss })`
  - `notifications`: array of `{ id, type, message }` objects.
  - `onDismiss(id)` callback to remove a notification.

- `Spinner({ size, color })`, `Modal({ open, onClose, title, children, width })`, `ProgressBar({ value, max, color, height })`, `Divider({ label })`
  - Small helper components — props are named clearly and used directly in the UI.

### Pages

Each page under `src/pages/` takes no props when mounted by `App.jsx`. They pull data and actions from the store hook `useAppStore()` and use UI primitives described above.
